# SM — 시료 관리 (Sample Management)

## 개요

시료(Sample)는 이 시스템의 가장 기본 단위다. 시스템에 등록된 시료만 주문 가능하며, 각 시료는 고유한 이름과 생산 속성을 가진다.

---

## 기능 목록

| ID | 기능 | 진입점 |
|----|------|--------|
| SM-01 | 시료 등록 | 메인 메뉴 → 시료 관리 → 1 |
| SM-02 | 시료 조회 | 메인 메뉴 → 시료 관리 → 2 |
| SM-03 | 시료 검색 | 메인 메뉴 → 시료 관리 → 3 |

---

## SM-01 시료 등록

### 입력값

| 항목 | 타입 | 제약 |
|------|------|------|
| 이름 | string | 필수, 비어있으면 거부 |
| 평균 생산시간 | int | 필수, 양수 (단위: 분) |
| 수율 | double | 필수, 0.0 초과 1.0 이하 |

### 처리 흐름

```
입력값 검증
    │
    ├─ 유효 → SampleRepository::create() 호출
    │             └─ ID 자동 부여, stock = 0 으로 초기화
    │             └─ samples.json 저장
    │             └─ 등록 완료 메시지 출력
    │
    └─ 유효하지 않음 → 오류 메시지 출력, 재입력 또는 취소
```

### 비고

- `stock`은 직접 입력하지 않으며 초기값 0으로 생성됨
- 이름 중복에 대한 별도 제약 없음 (ID로 구분)

## SM-02 시료 조회

### 출력 항목

| 컬럼 | 설명 |
|------|------|
| ID | 시료 고유 식별자 |
| 이름 | 시료명 |
| 평균 생산시간 | 단위: 분 |
| 수율 | 소수점 표시 (예: 0.90) |
| 재고 | 현재 stock 수량 |

### 처리 흐름

```
SampleRepository::findAll() 호출
    │
    ├─ 데이터 있음 → 테이블 형식으로 콘솔 출력
    └─ 데이터 없음 → "등록된 시료가 없습니다" 메시지 출력
```

---

## SM-03 시료 검색

### 입력값

| 항목 | 타입 | 제약 |
|------|------|------|
| 검색어 | string | 필수, 시료 이름 대상 부분 일치 검색 |

### 처리 흐름

```
검색어 입력
    │
    └─ SampleRepository::findAll() 후 name 포함 여부 필터
            │
            ├─ 결과 있음 → SM-02 와 동일한 형식으로 출력
            └─ 결과 없음 → "검색 결과가 없습니다" 메시지 출력
```

## 관련 엔티티

```cpp
struct Sample {
    int         id                = 0;
    std::string name;
    int         avgProductionTime = 0;   // 분
    double      yield             = 0.0; // 0.0 ~ 1.0
    int         stock             = 0;
};
```

## 관련 Repository

- `SampleRepository::create()` — SM-01
- `SampleRepository::findAll()` — SM-02, SM-03
