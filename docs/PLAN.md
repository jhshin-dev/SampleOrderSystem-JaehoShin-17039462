# PLAN — Phase별 개발 계획

각 Phase는 독립적으로 빌드·실행 가능한 SW를 목표로 한다.
고객님은 각 Phase 완료 후 직접 실행하여 "고객님 테스트 포인트"를 중심으로 확인하고 피드백을 주시면 됩니다.

---

## Phase 1 — 메인 메뉴

### 개발 목표

프로그램을 실행하면 메인 메뉴가 출력된다. 아직 어떤 기능도 구현되지 않으므로 메뉴 항목을 선택하면 "준비 중" 메시지를 출력하고 메뉴로 돌아온다.

### 구현 범위

- 역할 선택 루프 (0 입력 시 종료)
- 역할 선택 후 각 역할 메뉴 출력 (하드코딩 — 시료: 0종, 총 재고: 0개)
- 역할 메뉴에서 1~4번 선택 시 "준비 중입니다" 출력 후 복귀
- 0 입력 시 역할 선택 화면으로 복귀

### 고객님 테스트 포인트

1. 프로그램 실행 시 역할 선택 화면이 출력되는지 확인
   ```
   ========================================
     반도체 시료 생산주문관리 시스템
   ========================================
    1. 주문 담당자
    2. 생산 담당자
    0. 종료
   ```
2. 1 선택 시 주문 담당자 메뉴, 2 선택 시 생산 담당자 메뉴가 출력되는지 확인
3. 각 메뉴에서 1~4 선택 시 "준비 중" 안내 후 해당 역할 메뉴로 복귀하는지 확인
4. 0 입력 시 역할 선택 화면으로 돌아가는지 확인
5. 역할 선택에서 0 입력 시 프로그램이 정상 종료되는지 확인
6. 잘못된 입력(예: 9, 문자) 시 오류 메시지 후 메뉴가 재출력되는지 확인

## Phase 2 — 시료 등록

### 개발 목표

시료를 시스템에 등록할 수 있다. 등록한 데이터는 JSON 파일에 저장되어 프로그램 재시작 후에도 유지된다.

### 구현 범위

- `Sample` 구조체, `IRepository<T>`, `JsonRepository<T>`, `SampleRepository`
- SM-01 시료 등록 (이름·평균 생산시간·수율 입력, ID 자동 부여, stock=0 초기화)
- `data/samples.json` 자동 생성·저장·로드

### 고객님 테스트 포인트

1. 시료 관리 → 등록 선택 후 이름·평균 생산시간·수율을 입력하면 "등록 완료" 메시지가 출력되는지 확인
2. 프로그램을 종료했다가 재실행해도 등록한 시료가 **사라지지 않는지** 확인
3. 수율에 0 이하 또는 1 초과 값 입력 시 **오류 메시지**가 출력되는지 확인

---

## Phase 3 — 시료 조회 · 검색 + 요약 정보 갱신

### 개발 목표

등록된 시료 목록을 확인하고 이름으로 검색할 수 있다. 메인 메뉴 상단의 요약 정보(시료 수, 총 재고)가 실제 데이터를 반영한다.

### 구현 범위

- SM-02 시료 목록 조회 (ID·이름·평균 생산시간·수율·재고 표시)
- SM-03 시료 이름 검색 (부분 일치)
- 메인 메뉴 요약 정보 동적 계산

### 고객님 테스트 포인트

1. 시료 목록에 **등록한 모든 시료**가 올바른 항목으로 출력되는지 확인
2. 이름 일부를 입력했을 때 **해당 시료만 필터링**되는지 확인
3. 검색 결과가 없을 때 **"검색 결과가 없습니다"** 메시지가 출력되는지 확인
4. 메인 메뉴로 돌아오면 요약 정보의 **시료 수가 실제 등록 수**와 일치하는지 확인

---

## Phase 4 — 주문 접수

### 개발 목표

고객 주문을 접수하여 RESERVED 상태로 등록할 수 있다. 접수된 주문 목록도 확인할 수 있다.

### 구현 범위

- `Order` 구조체, `OrderStatus`, `OrderRepository`
- OR-01 주문 접수 (시료 ID·고객명·수량 입력, 초기 상태 RESERVED)
- OR-02 접수 주문 목록 (RESERVED만 표시)
- `data/orders.json` 자동 생성·저장·로드

### 고객님 테스트 포인트

1. 주문 접수 시 시료 ID·고객명·수량을 입력하면 **RESERVED** 상태로 등록되는지 확인
2. 접수 주문 목록에서 방금 접수한 주문이 **시료명·고객명·수량·접수일시**와 함께 출력되는지 확인
3. 존재하지 않는 시료 ID로 주문 시 **오류 메시지**가 출력되는지 확인
4. 수량에 0 이하 값 입력 시 **오류 메시지**가 출력되는지 확인

---

## Phase 5 — 주문 승인 · 거절

### 개발 목표

접수된 주문을 승인 또는 거절한다. 승인 시 **재고 상태에 따라 자동으로 CONFIRMED 또는 PRODUCING**으로 분기되는 핵심 비즈니스 로직이 동작한다.

### 구현 범위

- OR-03 주문 승인 (재고 충분 → CONFIRMED, 재고 부족 → PRODUCING)
- OR-04 주문 거절 (RESERVED → REJECTED)
- `isValidTransition()` 상태 전이 검증

### 고객님 테스트 포인트

1. **재고가 충분한** 시료의 주문을 승인 → **CONFIRMED**로 전환되는지 확인
2. **재고가 부족한** 시료의 주문을 승인 → **PRODUCING**으로 전환되고 생산 등록 안내 메시지가 출력되는지 확인
3. 주문을 거절 → **REJECTED**로 전환되고 접수 목록에서 **사라지는지** 확인
4. 이미 승인된 주문(CONFIRMED)을 다시 거절 시도 시 **오류 메시지**가 출력되는지 확인

---

## Phase 6 — 모니터링

### 개발 목표

담당자가 시스템 전체 상태를 한눈에 파악할 수 있다. 주문 상태별 현황과 시료별 재고 상태(여유·부족·고갈)를 확인할 수 있다.

### 구현 범위

- MO-01 주문량 확인 (RESERVED·CONFIRMED·PRODUCING·RELEASED, REJECTED 제외)
- MO-02 재고량 확인 (시료별 재고 수량 + 여유·부족·고갈 상태 표기)

### 고객님 테스트 포인트

1. 주문량 화면에서 **REJECTED 주문은 표시되지 않고** 나머지 4개 상태별 건수와 목록이 맞는지 확인
2. 재고량 화면에서 **여유·부족·고갈** 상태가 올바르게 표기되는지 확인
   - 재고 0 → 고갈
   - 재고 < 유효 주문 합산 → 부족
   - 재고 ≥ 유효 주문 합산 → 여유
3. Phase 4~5에서 만든 주문 데이터가 **모니터링에 정확히 반영**되는지 확인

---

## Phase 7 — 출고 처리

### 개발 목표

CONFIRMED 상태의 주문을 최종 출고(RELEASED) 처리한다. 주문 생애주기의 마지막 단계가 완성된다.

### 구현 범위

- RL-01 CONFIRMED 주문 목록 조회
- RL-02 출고 실행 (CONFIRMED → RELEASED, RELEASED 삭제 불가 가드)

### 고객님 테스트 포인트

1. 출고 처리 화면에 **CONFIRMED 상태 주문만** 표시되는지 확인
2. 주문 선택 후 출고 실행 → **RELEASED**로 전환되고 출고 목록에서 사라지는지 확인
3. CONFIRMED가 아닌 주문 ID 입력 시 **오류 메시지**가 출력되는지 확인
4. 출고 완료 후 모니터링 화면에서 **RELEASED 건수가 증가**하는지 확인

---

## Phase 8 — 생산 현황 · 대기 큐

### 개발 목표

재고 부족으로 생산 중인 주문의 현황과 대기 목록을 확인할 수 있다. 수율 기반 실 생산량과 총 생산시간이 화면에 표시된다.

### 구현 범위

- PL-01 생산 현황 (PRODUCING 주문, 실 생산량·총 생산시간 계산 표시)
- PL-02 생산 대기 큐 (FIFO — updatedAt 오름차순, 생산 라인 등록 시각 기준)

### 고객님 테스트 포인트

1. 생산 현황에서 **실 생산량이 공식대로** 계산되는지 확인
   - 예) 부족분 60, 수율 0.9 → `⌈60 / (0.9 × 0.9)⌉ = 75개`
2. 생산 대기 큐가 **접수 순서(먼저 들어온 주문이 위)**로 정렬되는지 확인
3. 현재 생산 중 주문이 없을 때 **"생산 중인 주문이 없습니다"** 메시지가 출력되는지 확인

---

## Phase 9 — 생산 완료 처리

### 개발 목표

생산이 완료된 주문을 CONFIRMED로 전환하고 시료 재고에 반영한다. 전체 기능이 완성되어 End-to-End 시나리오가 동작한다.

### 구현 범위

- PL-03 생산 완료 처리: `PRODUCING → CONFIRMED` 자동 전이
- `Sample.stock += 실 생산량` 재고 갱신
- `samples.json` / `orders.json` 동시 저장

### 고객님 테스트 포인트

1. 생산 완료 처리 후 주문이 **CONFIRMED**로 전환되는지 확인
2. 해당 시료의 **재고가 실 생산량만큼 증가**하는지 확인
3. 모니터링에서 **재고 상태(부족 → 여유)가 갱신**되는지 확인
4. 생산 완료 후 출고 처리 화면에 해당 주문이 **CONFIRMED 목록에 나타나는지** 확인
5. Phase 1 ~ 9 전체를 연결한 시나리오 테스트
   - 시료 등록 → 주문 접수 → 승인(재고 부족) → 생산 완료 → 출고 처리 순서로 진행하여 모든 상태 전이가 정상 동작하는지 최종 확인
